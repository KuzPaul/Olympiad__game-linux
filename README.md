## О проекте

Студенты проходят пять тематических модулей в браузере: брифинг, таймер, интерактивные задания и фиксация результата. Преподаватель видит рейтинг участников и команд, управляет учётными записями и настройками занятия.

Один компьютер в классе — сервер, остальные подключаются по локальной сети. Запуск через Docker — без установки Python на каждое рабочее место.

## Модули

| | Тема |
|---|------|
| 1 | Классификация программного обеспечения |
| 2 | Задачи и процессы |
| 3 | Учётные записи и права доступа |
| 4 | Сетевые средства Linux |
| 5 | Bash: поэтапный сценарий и зомби-процессы |

## Возможности

- одна официальная попытка на модуль у студента, неограниченные проходы у тестировщика;
- рейтинг по участникам и по командам / факультетам;
- фоновая музыка с централизованным отключением для аудитории;
- сохранение результатов между перезапусками (данные в `docker-data/`).

## Роли

| Роль | Кто | Что может |
|------|-----|-----------|
| **Преподаватель** | `admin` | Панель управления, студенты, пароли, рейтинги |
| **Студент** | регистрация на сайте | Прохождение модулей, личный прогресс |
| **Тестировщик** | создаёт преподаватель | Отладка заданий без лимита попыток |

## Стек

Python · Flask · SQLAlchemy · SQLite · ES modules · Jinja2

## Быстрый старт

```bash
cp .env.example .env   # SECRET_KEY и ADMIN_INITIAL_PASSWORD
docker compose up --build
```

**http://127.0.0.1:5001** · логин преподавателя: `admin`
