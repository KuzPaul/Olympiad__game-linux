# Эталонные ответы

Файл `all_answers.json` генерируется из исходных тем в `topics/` и предназначен для **проверки заданий**, ревью и синхронизации с клиентом (в игре по-прежнему используются данные из `topics/*/topic.py`).

Обновить после изменения тем:

```bash
python3 scripts/export_answers.py
```

## Поля в записях по задачам (модули с типом `builder` и `zombie_script`)

Для каждого шага в массиве `answers` используется:

| Поле | Смысл |
|------|--------|
| `task_index` | Номер задания по порядку (с нуля). |
| `scenario` | Текст условия, как в теме. |
| `primary` | **Основной эталонный ответ** — строка или список токенов (в сетевом модуле список — это разбор команды по словам). Для темы «Зомби и скрипт» это обычно **одна или несколько строк bash-кода** в виде одной строки JSON (с `\n` если в эталоне несколько строк). |
| `accepted_variants` | **Допустимые варианты** того же задания (другой синтаксис с тем же смыслом: например другой shebang или пробелы внутри `$(( ... ))`). Если поля нет — допускается только `primary`. |

Итого: **`primary`** — главный правильный ответ для ревью; **`accepted_variants`** — не обязательны, но перечисляют эквивалентные записи, которые автопроверка на клиенте тоже засчитывает.
